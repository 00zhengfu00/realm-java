TightDB Java Reference

This reference manual documents the high-level Java interface to TightDB. The high-level interface is suited for general application development. The low-level interface is documented elsewhere.

In this reference manual, a cell means a specific column at a specific row.

Table Class

The Table class allows you to efficiently work with data in terms of rows and columns.

Note: Tables should be generated using the @Table annotation. The annotation for a Java class with name foo will generate a number of classes representing rows (Foo), queries (FooQuery), result sets (FooView), and the table (FooTable).

Example:

public class Example {

    // PersonTable, Person, PersonQuery, and PersonView are generated:

    @Table
    class person {
        String name;
        int age;
        boolean hired;
    }

}

Query Class

Query objects are used to build-up queries. The query classes are generated by the @Table annotation.

Query is build using a fluent interface (method invocations can be daisy chained). Conditions are by default folded using logical-and. Query objects can be reused, and queries can be repeatly executed by the find_all() method (resembles the prepare/execute pattern in SQL clients).
